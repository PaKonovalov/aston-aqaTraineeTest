- единый принцип работы в команде
- быстрый деплойменнт

    модели ветвления:
    1. git flow - есть несколько веток (master берегается от всего; develop, в которую вливаются все свежие
    изменения, в ей проверяется общая работоспособность кода далее, ветки разработчиков; может быть ветка hotfix,
    для внесения быстрых изменений)

    2. Trank Based Development (TBD) основана на единой ветке master, а ветки для отдельных задач создаются
    ненадолго и удаляются после вливания
