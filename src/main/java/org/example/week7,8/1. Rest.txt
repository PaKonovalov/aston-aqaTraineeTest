1. Теоритические знания (что такое Rest, SOAP, разница между ними)

REST API — это архитектурный подход, который устанавливает ограничения для API: как они должны быть устроены
и какие функции поддерживать.

Нужен для: - удобства, производительности

Слово REST — акроним от Representational State Transfer, «передача состояния представления»

REST API использует методы HTTP

в HTTP прописан набор действий, который можно описать аббревиатурой CRUD: create — «создать», read — «прочитать»,
update — «обновить», delete — «удалить».

GET для чтения, а PUT и PATCH — для разных видов обновления, DELETE - для удаления

REST API используют когда:

    - пропускная способность соединения с сервером ограничена;
    - нужно соединить мобильные приложения с серверными;
    - проект разбит на микросервисы;
    - звестно, что систему нужно будет масштабировать.

REST API поддерживает форматы: JSON, XML, TXT, CSV, HTML

Недостатки: сложнее обеспечить безопасность, труднее выполнить операции с сохранением состояния
