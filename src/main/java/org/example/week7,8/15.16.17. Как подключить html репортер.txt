    17. Аннотации allure:

        @Step - Аннотация, которой помечаются степовые методы. Значение - имя соответствующего степа, может
        содержать в себе переменные {method}, {0}, {1}, etc. - имя метода и его аргументы соответственно.
        По умолчанию значение {method}.

        @Attach - Аннотация, которой помечаются методы, сохраняющие attachments. Имя может содержать в себе
        переменные {method}, {0}, {1}, etc. - имя метода и его аргументы соответственно, тип может быть TXT, XML,
        HTML, PNG, JPG, OTHER. Метод должен возвращать java.lang.String или java.io.File. TXT, XML, HTML в отчете
        будут в iframe, PNG, JPG - в теге img. Для типа OTHER - только ссылка для скачивания файла. По умолчанию
        имя - {method}, тип - AttachmentType.TXT.

        @Title - Меняет имя теста.

        @Description - Добавляет описание.

        @Severity - С помощью этой аннотации изменить установить важность теста. Возможные значения - TRIVIAL,
        MINOR, NORMAL, CRITICAL и BLOCKER. По умолчанию - NORMAL.

        @EPIC, @FEATURE, @STORY - Набор аннотаций, предназначенных для того, чтобы сделать группировку тестовых
        примеров в дереве более гибкой и информативной. Аннотации соответствуют гибкому подходу к определению
        задач.
        Эти аннотации могут быть реализованы на уровне класса или на уровне метода.
        Epic определяет задачу самого высокого уровня, которая будет разбита на функции.
        FEATURE будут группировать конкретные истории, обеспечивая легко читаемую структуру.
        Поскольку STORY является низшей частью иерархии epic-feature-story, история на уровне класса добавляет
        данные ко всем методам класса.

        @OWNER - владелей теста
